SECTION 02 · CONCEPT PRIMER

What you should already grasp

🧠
MENTAL MODEL — HOLD THIS BEFORE THE SESSION
IPS is the 'tripwire' layer of UTM. On FortiGate you pick between Filter type (broad — pick severity/protocol/OS + target) and Signature type (surgical — one specific rule), each with actions (Block, Monitor, Default). Placement matters: IPS is usually turned on at the NGFW/DCFW that fronts protected servers.

The session will teach these ideas from the study guide pages named above. If any of the bullets below feel foreign, expect to slow down at that point in the session — otherwise you can move at pace.

  • IPS profile lives under Security Profiles > Intrusion Prevention
  • Two rule types: Filter (broad match by severity/protocol/OS/target — thousands of signatures at once) and Signature (single-CVE precision)
  • Actions: Block (drop + log), Monitor (log only), Default (per-signature default action)
  • Target: Server (protect servers), Client (protect endpoints), or both
  • Packet logging: capture packet for post-mortem; enable only for troubleshooting/false-positive handling — disk-expensive
  • Rate-based settings: catch protocol anomalies (e.g., N SYNs from one source in T seconds)
  • Use Case 1 (from the guide): Filter, action=Block, target=Server+Client, all severities/protocols/OS — very broad, ~10,320 signatures — but this scale of blocking risks false positives
