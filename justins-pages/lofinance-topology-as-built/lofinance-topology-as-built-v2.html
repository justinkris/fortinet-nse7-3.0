Redistribution

All redistribution defaults to disabled. Exactly one static is redistributed at the hub: the DR legacy management subnet 10.99.50.0/24, gated by access-list MGMT-ONLY with exact-match enable. Automatic trust = blast radius; the filter is the leash.

  • Fingerprint to remember: Full/ - everywhere on the tunnels — no DR/BDR elections on point-to-multipoint.
  • Backbone rule: every non-backbone area must connect to Area 0; DR2PAR exists partly so a HQ failure cannot partition the backbone.
  • Symmetry is your job, not OSPF's: cost is paid on exit; asymmetric costs across parallel paths break stateful firewalls (S16).

11 · The Control Plane

Central Management Model (Sessions 28–34)

FortiManager is the source of truth. The management model matured across Season 3 into a layered, auditable machine — the part of the topology that grew most while adding zero hardware after S28.

  • ADOM structure: Global ADOM above two local ADOMs (LoFinance 7.6 / Vaultline mixed). The Global ADOM distributes a shared six-rule baseline via assign→install (S30).
  • Normalized interfaces: the baseline references WAN — a normalized interface with per-platform mappings (100F→port1, 200F→port3, 60E→wan1) so one audited package resolves across every model (S31).
  • Dynamic objects & metadata: Branch_LAN dynamic object (30 per-device values) + $(branch_lan) metadata variable — thirty branches, one package, one value table (S32). HQ DNS static by design.
  • Scripts: fleet scripts (remote target) and device-DB scripts (delivered via install), every run logged in revision history — auditor-readable (S33).
  • Continuous CLI templates: fleet-wide enforcement at the device layer, overwrite-on-install, fed by metadata variables; change-control discipline in force: fix-winning-level → canary → preview → widen (S34).
  • Drift doctrine: managed boxes are layer-edited only; direct CLI on a managed FortiGate is drift requiring immediate retrieve; reconcile-before-install for drifted devices (S29–30).
  • Debt IPsec templates hands-on still owed — the crying cactus endures (S13, hard S18 commitment).

13 · How It Got Here

Topology Evolution, Session by Session

Sydney HQ only: Internet → edge FortiGate → core switch → server room, trading floor, exec floor, DMZ, management network, guest Wi-Fi.
