Interface Says Where.
Gateway Says How to Leave.

One minute. The member that was up all weekend and never carried a packet.

"Interface says where. Gateway says how to leave.
No gateway, no route, no probe, no member."
Zone stays optional. Interface + gateway are not.
The collapse — one missing field, five dominoes
no set gateway→ no member route→ probe can't leave→ 100% loss → state(dead)→ never eligible for any rule

The link light was on the whole time. Up is a fact. It is not a route.

config system sdwan
    config members
        edit 3
            set interface "port3"
            set zone "underlay"        ← optional
            set gateway 192.0.2.1      ← the one that was never typed
        next
    end
end
The rule does not create reachability. SD-WAN will not steer traffic to a member that has no valid route to that destination. A rule only chooses among members that already have one. Listing a member in priority-members proves nothing about whether it can get there.

Two commands that tell you instantly
CommandWhat a missing gateway looks like
get router info routing-table all Only a connected route for that interface. No auto-generated default via it.
diagnose sys sdwan health-check state(dead), packet-loss(100.000%), latency and jitter both 0.000 — the giveaway that nothing ever left, as opposed to a genuinely bad link.
Read the shape of the failure. A degraded link returns real numbers badly. A link with no path out returns zeroes. Zeroes across every metric are a configuration symptom, not a carrier symptom.
