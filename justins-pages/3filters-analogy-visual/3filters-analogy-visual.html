SESSION 22 · SECURITY PROFILES · THREE FILTERS, ONE JOB?

The LoFinance Mail Room

Three sorters stand at the same conveyor belt. Every envelope is a connection. Each sorter decides what to do with it — but each one looks at something completely different, and each one works in a different room.

WHAT EACH ONE SEES WHERE EACH ONE LIVES
SORTER № 1

Web Filter

“Read by the address on the front.”
WHAT IT SEES
Domains, URLs, SNI / SAN — the address written on the front
It reads where the envelope claims to be going.
Layer 7 — web Flow OR proxy
WHERE IT LIVES
Security Profiles
Attached to a firewall policy like the other profiles.
config webfilter profile
WHERE ITS KNOWLEDGE LIVES
Not on the FortiGate. It phones FortiGuard / FDS on-demand for rating lookups, then remembers recent answers in a cache.
SORTER № 2

Application Control

“Recognised by the sender's handwriting.”
WHAT IT SEES
Transmission patterns — app signatures & protocol decoders
Not where it's going — who wrote it. The app betrays itself by how it behaves on the wire.
Layer 7 — OSI Flow-based ONLY Rate-based IPS sigs
WHERE IT LIVES
Security Profiles
Also usable in traffic shaping — and SD-WAN app steering if enabled.
config application list
WHERE ITS KNOWLEDGE LIVES
Locally, as the Application Definitions database. Check it with diagnose autoupdate versions.
SORTER № 3

ISDB

“Sorted by postal district. Never opened.”
WHAT IT SEES
IP addresses (L3) + TCP/UDP ports (L4) — the postal district
A lookup against a list of known destinations. The envelope is never opened.
Layer 3 & 4 NO inspection
WHERE IT LIVES
Policy & Objects — it's an address object, not a profile
Used as a destination in firewall policies and SD-WAN rules.
config firewall internet-service-name
WHERE ITS KNOWLEDGE LIVES
Locally, as the Internet-service Full Database. Same check: diagnose autoupdate versions.

SEÑOR'S WHITEBOARD — SAY IT IN ONE BREATH

Web filter reads the address. Application control recognises the handwriting. ISDB sorts by district and never breaks the seal. Two are profiles; the third is an object. Two keep their knowledge on the box; the address-reader phones FortiGuard and keeps a cache.

⚠ RILEY'S TICKET — THE SEAM BETWEEN SORTERS

“Facebook is blocked but the Facebook APP still works?? — Trading Floor.” The web filter read the address on browser traffic and stopped it. But the app doesn't write an address the web filter recognises the same way — it has to be caught by its handwriting. One symptom, two sorters: knowing which one sees what is the whole ticket.
