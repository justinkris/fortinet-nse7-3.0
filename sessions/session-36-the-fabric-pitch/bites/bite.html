Security Fabric — Purpose & Architecture

Why it exists

Every standalone firewall's knowledge is an island. Session tables, device visibility, and logs are siloed per box. The fabric makes Fortinet products (and, via connectors, third parties) into one nervous system — so "show me every device" has an answer.

Three pillars — memorize verbatim: BIA

  • Broad — detect & enforce everywhere (endpoint → network → cloud).
  • Integrated — close gaps; products operate as one.
  • Automated — faster time-to-prevention; self-healing network.
Exam trapDistractors sound more architectural ("centralized, distributed, hybrid"). Pick Fortinet's boring official triad.

Shape & core

Tree topology. One root FortiGate = the most upstream device (position, not traffic volume). Downstream FortiGates point at their upstream. Root hosts: Physical/Logical Topology views, fabric connectors, authorization.

Core components = FortiGate + FortiAnalyzer. Topology = "what exists now"; FAZ = "what happened last Tuesday" — auditors live in last Tuesday.

Connectors

Fabric connectorsExternal connectors
Fortinet products: FortiGate, FAZ, EMS + Central Mgmt, Sandbox, FortiNAC/NDR familyEverything else: AWS/GCP, Kubernetes/VMware, FSSO/RADIUS, threat feeds

External feed: plain-text list at a URL → fetched via HTTP/HTTPS/STIX (or Push API) → becomes a resource object usable in firewall policies. List updates; policy never edited.

Logging laws

  • The fabric logs each session once — owned by the first FortiGate in the path.
  • Peers recognised by MAC address → no duplicate traffic logs.
  • Exception 1 — NAT: NATing member adds a traffic log (new src IP/port = evidence existing nowhere else).
  • Exception 2 — UTM: UTM events always log at the inspecting device.
  • Members log to FAZ directly & independently → root down ≠ logging stops.
  • FAZ correlates traffic + UTM logs automatically — zero extra config.

Joining

Downstream: role = Join Existing Fabric + upstream IP. Then it waits until an admin authorizes it on the root (System > Firmware & Registration for FortiGates; alert-icon notification for other products). Knowing the root's IP is never enough.

One-breath business case

"When — not if — we have an incident, regulators will demand a complete record of every device and session; the fabric turns that from a court date into a ten-minute report."
