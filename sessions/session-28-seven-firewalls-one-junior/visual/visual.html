🎨 SESSION 28 · VISUAL LEARNING

The FortiManager Building

A three-floor cutaway that turns FortiManager's architecture into a place you can walk through — global penthouse, walled tenant floors, and the device-copy vault below.

Recommended style

Layer-by-Layer Breakdown fused with a Config Journey. This session's core idea is a strict three-level hierarchy with directional flows (retrieve up, install down, assign down) and one hard isolation wall between tenants. A cutaway "building" makes the hierarchy spatial, the arrows literal, and the wall visible — exactly the mental model to burn in. Below are the image placeholders; copy each prompt into your image generator.

5 · The Global Test — Everyone, or Just Them?
The when-to-go-global decision, and why all-global collapses the wall.
⚖️

Image generation prompt
Flat minimalistic detailed vector illustration, warm cream background (#faf5e9), soft-brown outlines, no gradients, no 3D. A sorting/decision scene shaped like a fork in a conveyor belt. A single object enters from the left on a conveyor, reaching a signpost that asks: "TRUE FOR EVERYONE?" Two chutes branch from it. TOP CHUTE labelled "YES -> GLOBAL DATABASE": three objects (syslog, NTP, DNS icons) ride up into a soft-blue penthouse tray at the top, each getting a small coral "g-" tag, then fanning out via two "ASSIGN" arrows down into two ADOM rooms below. Colour palette: cream, coral, muted gold, sage green, soft brown outlines, ink-navy text. Clean, calm, legible like an educational flow diagram. DO NOT include: photorealism, gradients, glossy 3D, real logos, dense text. The "everyone vs just them" fork must be the visual focus; the crumbling-wall inset should be small and clearly a warning.

Beside the global-database section, near the "when NOT to go global" note.
