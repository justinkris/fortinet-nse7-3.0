SESSION 41 · STUDY BITE · 5 MINUTES

The Fast Path

Session offloading in five minutes — mandatory this week (consolidation gap, per the register).

1 · The Rule of Three Verbs

Birth, Life, Death

CPU witnesses the birth — TCP: the whole three-way handshake goes through the CPU; UDP: only the first packet. The CPU does policy + routing + session creation, then installs the session key in the NP.

NP carries the life — subsequent packets skip routing, skip the kernel, never touch the CPU.

CPU handles the death — FIN/RST always punt to the CPU: teardown = delete kernel entry + uninstall NP key.

2 · The Four Disqualifiers

Who Doesn't Ride

  1. auto-asic-offload disable on the firewall policy (default: enable).
  2. Proxy-based profiles — and a mixed flow+proxy policy offloads nothing. One proxy profile poisons the whole policy.
  3. Session helper required (FTP is the classic).
  4. Tunnels — SSL VPN, GRE… except IPsec, which the NP handles natively. Say it as a pair: "tunnels DON'T — except IPsec DOES."
3 · The Evidence

Read the Confession

diagnose sys session list
npu info: flag=0x81/0x91, offload=9/9, ...     ← 9=NP7; slash = originate/reply
npu_state=0x000001 no_offload
no_ofld_reason: disabled-by-policy             ← FortiOS tells you WHY

9/9 = both directions on hardware. 9/0 = replies limping through the CPU.

4 · The Trap That Cost You Q6

Config Is the Future — the Session Table Is the Present

Offload is decided once, at session creation. Existing sessions never re-qualify after a config fix. Long-lived sessions stay slow until you kill them:

Remediation
diagnose sys session filter dstip <server>
diagnose sys session clear

Reconnect → new handshake → key installs → verify offload=9/9.

5 · Offloaded Packet Journey

What Gets Skipped

NP ingress → IP integrity check → (IPsec decrypt) → skips routing + kernel + CPU → (shaping) → (IPsec encrypt) → egress. The session key is the pre-computed routing+policy answer, replayed per packet.
