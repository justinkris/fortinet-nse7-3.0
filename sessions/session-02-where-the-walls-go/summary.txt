TECHNICAL CONCEPTS COVERED
--------------------------
1. Edge vs internal inspection are DIFFERENT JOBS: edge = content threats, deep
   inspection (NGFW duties); internal = lateral movement, least-privilege segmentation,
   low latency (ISFW duties). Student derived this split unprompted via threat model.
2. "More inspection everywhere" is the junior answer; right inspection at the right
   point is the senior answer. Latency/performance trade-off at internal points.
3. ISFW: FortiGate 100–900 (campus), 1–100 Gbps; segmentation/micro-segmentation
   (VLANs, VDOMs, zones), low latency, HW SSL inspection, integrated wireless LAN.
4. DCFW: FortiGate 1000–7000 (data centre), 10–1000 Gbps, hyperscale; east-west traffic
   dwarfs north-south (replication, backups, market-data fan-out); typically bundles
   multiple physical switch links into one logical interface (LAG) for redundancy +
   speed; segmentation via VRFs/VDOMs/zones; often protected by an NGFW gateway in
   front, though it can face the internet directly using VDOMs.
5. DEFW: FortiGate 30–90 (branch), up to 1 Gbps; firewall + IPsec VPN to HQ +
   segmentation + integrated W/LAN; NOT for deep analysis; backhaul model puts heavy
   inspection at HQ; if branch needs real local inspection (local internet breakout,
   growth), step up to 100–900 midrange.
6. CORE INSIGHT: NGFW is a ROLE, not a model. Spans 100–7000 + VM/cloud (1–40 Gbps).
   Three acronyms are PLACES (branch/campus/data centre); one is a JOB. An ISFW with
   UTM profiles enabled IS an NGFW. Same box, same invoice.
7. Hybrid mesh firewall: mixed models/roles distributed across the network, one FortiOS,
   one management console (FortiManager). Cloud family name-dropped: FortiGate VM,
   FortiCNF, FortiSASE (deep dive later).
