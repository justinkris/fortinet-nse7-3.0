⏱ 5-MINUTE BITE · SESSION 12 · ROUTE-BASED IPSEC BUILD

Tunnel Vision

The one idea

The tunnel is an INTERFACE.

Creating a phase1-interface makes FortiOS create a virtual interface. Packets enter the tunnel by winning a route to it and pass by matching a policy on it — the same machinery as any physical port. So "tunnel up" ≠ "traffic flows."

The five-item checklist

1. Phase 1
handshake, public IPs
2. Phase 2
SAs + selectors
3. Tunnel IPs
optional; routing later
4. Route
remote subnet → tunnel
5. Policies
both directions!

IKE is local-in traffic — the tunnel forms with zero forward policies. Policies govern what rides inside.

Replies free · initiators pay

Packet A (a reply): matches an existing session → passes with no reverse policy. Packet B (a new connection from the far side): first packet of its life → policy lookup → no match → implicit deny, policy 0, silent drop.

Write policies per direction of initiation, not direction of travel. Two-way site = both directions, both firewalls.

The build, compressed

config vpn ipsec phase1-interface
  edit "ParraTunnel-ISP1"
    set interface "wan1"
    set ike-version 2
    set network-overlay enable
    set network-id 1        ← twin tunnel gets 2; must match both ends
    set remote-gw 203.0.113.45
    set psksecret ***
config vpn ipsec phase2-interface
  edit "ParraTunnel-ISP1"
    set phase1name "ParraTunnel-ISP1"
    set src-subnet 10.1.0.0/24
    set dst-subnet 10.2.0.0/24
config router static
  edit 0
    set dst 10.2.0.0/24
    set device "ParraTunnel-ISP1"   ← NO gateway (point-to-point)

IKEv2's first phase 2 rides IKE_AUTH (the encrypted exchange) — hence no quick mode, four messages total.

Verify · convict

diag vpn ike gateway list        → version: 2, network-id: 1, state: established
diag debug flow …                → "find a route … via ParraTunnel-ISP1"  (routing OK)
                                   "Denied by forward policy check (policy 0)"
Policy 0 is not a policy anyone wrote. It's the implicit deny — a policy is missing. One-way-silent tunnel = missing internal→tunnel at the source site OR missing tunnel→internal at the far site. Debug closest to the source first.

Junior's mistake

Mirrored phase 1, phase 2, and the route — then wrote one policy, trusting statefulness to cover the other direction. Statefulness covers replies, not initiators. Marcus was not a reply.
