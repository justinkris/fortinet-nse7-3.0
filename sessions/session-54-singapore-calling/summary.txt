==================================================================
TECHNICAL CONCEPTS COVERED (EF pp. 132-136)
==================================================================
1. Why OSPF cannot cross administrative boundaries: link-state protocols share the FULL
   topology database with every neighbour and compute an identical, non-negotiable SPF result.
   No mechanism exists to share a partial map or exert per-route policy.
2. BGP as policy-based routing between independently-administered domains: advertise only what
   you choose, accept only what you choose. "Routing as policy, not routing as geometry."
3. Autonomous System (AS) definition: a set of routers/networks under one consistent
   administration, free to run any IGP internally (OSPF unaffected), identified externally by
   one AS number.
4. BGP is a PATH VECTOR protocol (contrast with OSPF's link-state): no shared map; routes carry
   an AS_PATH — the ordered list of AS numbers the route has traversed.
5. Loop prevention mechanism: a BGP router rejects any route whose AS_PATH already contains its
   own AS number. Automatic, per-route, requires no synchronised database.
6. AS types (three): Stub (single AS connection, local traffic only), Multihomed (multiple AS
   connections, but still only local traffic), Transit (routes local AND transit traffic —
   traffic neither originating nor terminating locally; ISP is the textbook example).
7. LoFinance becomes Multihomed once peering with Singapore + London + an ISP — NOT
   automatically Transit. Multihoming is about NUMBER of external connections; transit is about
   WHETHER traffic that isn't yours flows through you for someone else's benefit.
8. The "accidental transit AS" trap: AS_PATH alone does NOT prevent LoFinance from
   unintentionally advertising Singapore's routes onward to London (or vice versa). This
   requires a SEPARATE, deliberate defence: prefix/route filtering on outbound advertisements.
   Exam trap: don't confuse loop prevention (AS_PATH, automatic) with transit prevention
   (filtering, configured).
9. Private AS number range: 64512-65534 (IANA-reserved), directly analogous to RFC 1918
   private IPv4 space — usable for privately-agreed inter-organisation BGP without registry
   involvement or collision risk with real internet ASes (until/unless connected to the public
   internet without translation).
