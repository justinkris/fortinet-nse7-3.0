SSL Inspection — Envelope vs Letter

The model

Cert inspection = read the envelope (SNI + certificate, no decryption). Full inspection = open the letter (decrypt via authorised MITM).

Who works blind?

Web filter + app control: fine on the envelope. AV / IPS / DLP: blind without full inspection. App control names the app, never the action.

Mode ≠ depth

Full inspection runs in flow AND proxy mode. Proxy-only: SSL proxying + protocol port mapping.

The MITM three

Forged server cert · CA in client trusted stores · CA private key on the FortiGate. Warnings on all sites = CA missing.

SNI vs SAN

SNI (ClientHello): many certs, client picks — "separate cert per domain." SAN (in the cert): one cert, many names — wildcards allowed.

HTTPS protection under cert inspection

Limited — never "None." Bad certs and bad destinations still caught.
