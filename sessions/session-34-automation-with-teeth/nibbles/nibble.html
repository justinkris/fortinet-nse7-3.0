The one lineAssign stages. Install ships. Fix at the level that wins, verify at the smallest blast radius.
CLI templateA stencil above the device DB that overwrites it at install. Can't out-edit from below.
AssignA binding verb, not delivery. Flips device-layer + provisioning-template Modified. Ships nothing.
Pre-run vs continuousPre-run auto-unassigns (onboarding). Continuous re-overwrites every install (enforcement = the teeth).
Metadata: value vs referenceValue in Metadata Variables (dictionary). Reference $(name) in the template. The $ picker = seatbelt.
Policy-package scriptGreen = ran, not shipped. Flips package Modified. Install Policy Package to deliver.
Two-hop installpolicy layer → device database → FortiGate. Nothing skips the device DB — the single door.
Attached vs unattachedInstall carries what the package uses (referenced). Orphans stay in the ADOM DB.
ReinstallSkips wizard, keeps preview. Also drags staged device-DB mods down — preview catches them.
Exam heuristicTwo options both work? Pick the one still true in six months. permanent/drift = template; set-once = script.
