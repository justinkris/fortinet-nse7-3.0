Automation With Teeth

CLI templates, policy-package scripts, and the discipline that keeps power from becoming a blast radius.

Read this before work or before the exam. The five ideas that matter.
Idea 1 — The Stencil

A CLI Template Overwrites

A CLI template is a provisioning template — a stencil that sits above the device database and overwrites it at install. The template wins. You cannot out-edit it from below: a manual fix on the FortiGate is re-overwritten on the next install. Assigning one flips two statuses (device layer + provisioning template); the policy package stays quiet.

📌
Anchor
Assign stages. Install ships. Thirty amber triangles = staged, not live. Fix the stencil, not the glass.
Idea 2 — Two Flavours

Pre-Run vs Continuous

Pre-run: one-time onboarding (ZTP/LTP), auto-unassigns after install — fire once, gone. Continuous: stays bound, re-overwrites on every install — drift enforcement. That's the "teeth." Junior's typo recurred because his template was continuous.

Exam tell: "enforce a standard fleet-wide / auto-correct drift" → continuous template. "Set it once" → script.
Idea 3 — The Bent Pointer

Metadata: Value vs Reference

Value lives in Policy & Objects > Advanced > Metadata Variables (the dictionary; per-device mapping). Reference — $(dmz_name) — is used inside the template. The $ picker prevents misspelling the reference; hand-typing declines that seatbelt. Either a wrong value or a wrong reference hits all thirty — both are single shared objects.

The dictionary is right; the pointer is bent.
Idea 4 — The Single Door

Policy-Package Script & Two-Hop Install

A policy-package script that "ran" (green) flips the policy package Modified — the object is not live yet. Install Policy Package ships it: policy layer → device database → FortiGate. Nothing skips the device DB — it's the single door every change walks through. Only objects a policy references get carried; orphans stay in the ADOM DB.

Idea 5 — Reinstall

Skips the Wizard, Keeps the Preview

Reinstall bypasses the wizard (device + package already paired) but keeps the preview. It matters more because reinstall also drags staged device-DB modifications down the same door — the preview is where you catch them.

🔍
Anchor
"Reinstall skips the questions, never the confession."
The whole episode: Assign stages. Install ships. Fix at the level that wins, verify at the smallest blast radius, widen only after evidence.
