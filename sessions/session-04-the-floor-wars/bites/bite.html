SESSION 04 · BITE · VLAN DESIGN & TAGGING

The Floor Wars — 5-Minute Revision

The Problem

No VLANs = one broadcast domain = every device can directly frame every other device, no inspection point possible. This is always true regardless of switch model or physical layout.

Tagging Rules

  • 802.1Q = single tag, EtherType 0x8100, 4,094 VLANs.
  • 802.1ad (QinQ) = double tag, EtherType 0x88A8 (S-tag) + 0x8100 (C-tag), 16M combos.
  • FortiGate never tags on ingress — the preceding device (switch access port) always does that.
  • FortiGate can change a tag when routing between VLANs (strip on ingress, add on egress). Switches cannot alter a tag, only add/remove it unchanged.
Trap: tag is a property of the wire, not the packet — don't picture it as glued on permanently.

Config Skeleton

config system interface
  edit "VLAN20-Trading"
    set interface "internal1"
    set vlanid 20
    set ip 192.168.20.1 255.255.255.0
  next
end

VLAN interface = physical/aggregate interface (which wire) + vlanid (which tag). Works on physical, aggregate/LACP, FortiLink, hardware/software switch, and SSID interfaces too.

Real Segmentation = Gateway on FortiGate

VLANs alone ≠ security boundary. Gateway must live on FortiGate so inter-VLAN traffic is forced through a policy decision. Gateway on a switch = "filing cabinet with labelled but unlocked drawers."

Verify First

diagnose sys session list — no session at all = Layer 2 problem (tag/trunk/ARP), not the policy. Session present with a policy_id = policy problem (order, srcintf/dstintf).

Don't debug policy logic before confirming the traffic ever arrived as a routable frame.
