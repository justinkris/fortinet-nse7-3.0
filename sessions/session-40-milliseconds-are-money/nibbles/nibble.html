SPU Architecture: The Triggers

The Cut

NP moves packets (at the ports — traffic offload, IPsec, IP integrity check). CP reads packets (beside the CPU — pattern matching, SSL deep-inspect crypto). SoC/SP5 (NP7lite) = both, lighter, lower throughput.

The Number

NP7 = 12M sessions, capped by processor memory, overflow → CPU. Tables are per chip — distribute traffic across NP7s. Cabling is capacity planning.

The Architecture

NP Direct = interfaces wired straight to the NP, ISF eliminated, lowest latency. NP6: 2+ chips (200E/2000E/2500E). NP7: ultralow-latency ports X5–X8 (400F/600F/900G).

The Rule

Same chip in, same chip out — there is no bridge anymore. Offloaded traffic enters and exits on the same NP.

The Commands

diagnose hardware lspci | grep 1a29 → 4e36=NP6, 4e37=NP7. get hardware status → ASIC version: CP9 (discrete) or SOC5 (SP5 box).

The Trap

ISF-hop latency ≠ CPU-overflow latency. Different diagnoses, different fixes. And an NP-only box cannot carry heavy SSL deep inspection — that crypto is CP work.
