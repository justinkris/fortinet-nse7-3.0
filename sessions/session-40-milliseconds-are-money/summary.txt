TECHNICAL CONCEPTS COVERED (pp. 280–285)
------------------------------------------------------------------
1. SPU = Fortinet ASIC umbrella term; an SPU can be an NP, a CP,
   or both; offloads resource-intensive work from main CPU.
2. NP (latest NP7): works at the INTERFACE LEVEL; offloads/
   accelerates traffic; IPsec encrypt/decrypt for supported algos;
   IP integrity header check + early inspections (ACL/HPE handled
   at NP per life-of-packet, taught lightly). "Stationed at the
   ports."
3. CP (latest CP10; 900G carries CP9): "coprocessor for the CPU" —
   never in the packet path alone; pattern matching for flow-based
   UTM/IPS/app-ID/AV, SSL/TLS encrypt-decrypt for deep inspection,
   IPsec for supported algos. "Chained to the CPU's desk."
4. NP7 session limit: up to 12 MILLION sessions, limited by
   PROCESSOR MEMORY. Over-limit sessions go to the CPU (slow
   path). Ports are wired to SPECIFIC NP7s; tables are per-chip
   not per-box → must know interface-to-NP mapping and distribute
   incoming traffic across NP7s.
5. NP DIRECT architecture: removes the ISF (internal switch
   fabric — the hardware switch gluing front ports to
   processors); interfaces connect DIRECTLY to the NP; lowest
   forwarding latency. Availability: NP6 boxes with 2+ NP6
   (200E/2000E/2500E); NP7 boxes with ultralow-latency interfaces
   (400F/600F/900G — ports X5–X8 direct to NP7).
   REQUIREMENT: offloaded traffic must ENTER AND EXIT through
   interfaces on the SAME NP6/NP7. Student's rack-tape rule:
   "Same chip in, same chip out — there is no bridge anymore."
6. SoC/SP family: entry/mid-range; combines light NP + light CP
   on one chip with LOWER THROUGHPUT. Latest SoC4 and SP5; SP5
   alias = NP7lite (appears in CLI). 91G shows SOC5 in ASIC field.
7. Identification CLI:
   - diagnose hardware lspci | grep 1a29 → 1a29 = Fortinet vendor
     ID; NPID 4e36 = NP6, 4e37 = NP7.
   - get hardware status → ASIC version field = CP generation
     (e.g. CP9 on 900G) or SOC5 (confirms SP5) on branch boxes.
8. Distinction drilled: ISF-latency vs CPU-overflow-latency are
   DIFFERENT diagnoses with different fixes.
