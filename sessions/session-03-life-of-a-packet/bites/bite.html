Session 03 · 5-Minute Bite

Life of a Packet

The Core Idea

Cheap checks first, expensive checks last. Marcus's order proves this: FortiGate does ACL / HPE / IP header checks at the interface (cheap, hardware, fast) before spending IPS/AV/SSL-decrypt cycles (expensive) on it.

Two Chips, Two Jobs

NP (network processor) = forwarding specialist, works at the interface, wire-speed. CP (content processor) = content specialist, does IPS pattern matching / SSL decrypt / AV. Entry-level boxes combine both into one SP chip.

Session Setup vs Fast Path

  • TCP: whole 3-way handshake through the CPU. UDP: just the first packet.
  • Session key installed in NP → rest of the session fast-paths, skips CPU.
  • FIN/RST always go to the CPU — session teardown is CPU-owned, no matter how long it was offloaded.

The Pipeline Order

DNAT → Routing/RPF/SD-WAN → Policy Lookup → (UTM/CP inspection) → SNAT

Why: DNAT resolves the real destination before routing can pick an interface. Policy needs the resolved interface. SNAT happens last, after policy already decided.

Segmentation Is Real Because...

DNAT can be a no-op internally (no VIP needed). Policy lookup is never a no-op — implicit deny applies to every packet, everywhere. That's the mechanism that makes segmentation enforceable, not cosmetic.

Trap to remember: "established session" questions are about what happens after setup (FIN/RST), not the handshake that created it.

Two Commands

get hardware status → model + ASIC version.
diagnose hardware lspci | grep 1a29 → which chip is on which interface (needed on multi-NP boxes).
