TECHNICAL CONCEPTS COVERED (pp. 89-92)
------------------------------------------------------------------
1. FGCP ETHERTYPES (FGCP is Ethernet-typed, NOT IP):
   - 0x8890 heartbeat in NAT/route mode
   - 0x8891 heartbeat in transparent mode
     Both carry: neighbour discovery within the HA group, primary
     election, failure detection, traffic redistribution, other sync.
   - 0x8893 configuration synchronisation.
   Sniff: diagnose sniffer packet <hbdev> "ether proto 0x8890" 4
   CRITICAL: sniff on the HBDEV, not a data port. Junior's fault.

2. SESSION SYNCHRONISATION PATH — the episode's core mechanism.
   DEFAULT: session sync travels over the HA HEARTBEAT LINK.
   Consequence: bulk session-table data competes with small,
   latency-critical heartbeat frames. Under load (market open),
   heartbeats are delayed/dropped → partner presumed dead →
   FALSE FAILOVER. Cluster is not broken; it is deafened by its own
   chatter.
   FIX: dedicated session-sync interfaces.
     config system ha
         set session-pickup enable
         set session-sync-dev "port5" "port6"
     end
   - Multiple interfaces listed → sync traffic is LOAD BALANCED
     across them.
   - If ALL dedicated sync interfaces fail → session sync REVERTS to
     the heartbeat link. Graceful, but silently restores the original
     failure mode with no alarm. (Q4 trap.)

3. VIRTUAL CLUSTERING:
   - FGCP extended to a cluster whose members have VDOMs enabled.
   - ACTIVE-PASSIVE ONLY. Never a-a. (`set mode a-p`)
   - Max 30 virtual clusters.
   - Max HA group-id = 7 (standard HA allows higher) — exam detail.
   - All other cluster requirements identical to standard HA.
   - The conceptual key: "primary" stops being a property of the BOX
     and becomes a property of the VDOM.

4. VDOM PARTITIONING:
   - One member is primary for some VDOMs, the other member primary
     for others. Both boxes carry real traffic simultaneously while
     the cluster remains active-passive.
   - Selection mechanism = `set priority` PER VCLUSTER. Highest
     priority wins that vcluster. Priority is used because it is the
     only administratively assigned election criterion (interface
     state and uptime are facts; serial is factory-stamped).
   - root VDOM must be assigned to a vcluster.
   - FAILOVER: the surviving member processes ALL traffic for ALL
     VDOMs. Therefore each member must be sized for 100% of total
     load. VDOM partitioning gives UTILISATION, not CAPACITY.

   Config taught (both members, mirrored priorities):
     config system ha
         set mode a-p
         set vcluster-status enable
         config vcluster
             edit 1
                 set override disable
                 set priority 200      (NGFW-1) / 100 (NGFW-2)
                 set vdom "VDOM1" "root"
             next
             edit 2
                 set override disable
                 set priority 50       (NGFW-1) / 200 (NGFW-2)
                 set vdom "VDOM2"
             next
         end
     end

5. WHY NOT ACTIVE-ACTIVE (carried from S45): in a-a the primary owns
   the vMACs, so every packet lands on the primary FIRST and selected
   sessions take an extra hop out to the secondary and back. It does
   not move the front door — which was exactly Penny's problem.

TROUBLESHOOTING APPROACH TAUGHT
------------------------------------------------------------------
Symptom (flapping, time-correlated) → check WHERE you are listening
before concluding silence → confirm heartbeat is alive → ask what
ELSE shares that wire → correlate failover timestamps against load
windows → fix by separating control plane from data plane.
Central lesson: THE TIMING DETAIL IS THE DIAGNOSIS. "Only at market
open" eliminates override, priority, and interface-state causes
instantly, because none of them are time-of-day dependent.

CONFIGURATION DECISIONS
------------------------------------------------------------------
- Dedicated session-sync links added (port5/port6), load-balanced.
- session-pickup enabled.
- Virtual clustering enabled; vcluster 1 = Trading + root (primary
  FGT-1); vcluster 2 = Compliance (primary FGT-2).
- override left DISABLED on both vclusters.
- Both members sized for 100% of total load (design constraint
  accepted by Penny).
