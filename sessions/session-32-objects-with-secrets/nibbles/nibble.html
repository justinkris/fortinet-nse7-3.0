One object, many truths. Generic object in the package → per-device value in FMG → substitution at install. The FortiGate gets the answer, never the question.
Metadata variables: Policy & Objects > Advanced. Magnifying-glass-$ icon. Type $ in a field. Default + per-device override — per-device wins. Extra power: works in CLI/provisioning templates too.
Unresolved at install = ABORT. Missing mapping · WAN≠wan · unfilled variable — one law: install validates, fails loudly, pushes nothing. Empty default = deliberate onboarding tripwire.
Variables are for values that vary. Identical everywhere (HQ DNS)? Plain static object. Needless indirection = pure cost.
Security profiles can't per-device map (nor services/schedules) → unique names required. Importing per-box differences of unsupported types = conflict → rename + re-import.
