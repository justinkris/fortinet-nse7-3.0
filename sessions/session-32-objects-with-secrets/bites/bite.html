Objects with Secrets

The machine (same as Friday, aimed at objects)

Generic object in the package → per-device value stored in FMG against each device → substitution at install. The FortiGate receives only the resolved value. The box gets the answer, not the question.

One address object Branch_LAN = 172.16.1.0/24 on Branch1, 192.168.0.0/24 on Branch2 — thirty branches, one certified package, one value table.

Metadata variables — the general weapon

  • Created at Policy & Objects > Advanced > Metadata Variables.
  • GUI icon: magnifying glass + $. In a field, type $ → popup; + to create.
  • Default value + per-device overrides. Per-device wins — specific beats general.
  • Extra reach vs dynamic objects: usable in provisioning/CLI templates — any templated config line can vary per box, not just an address. Template + $ variable can overwrite both layers; assignment flags statuses Modified.

Unresolved value at install

No default + no per-device value = install FAILS for that device. Nothing pushed. Install is a validation gate, not a delivery van — fail loudly beats succeed wrongly. One law, three costumes: missing mapping, case mismatch (WAN ≠ wan), unfilled variable → all abort.

An empty default is a tripwire feature: it forces every onboarding to consciously fill in the branch value.

When NOT to abstract

Variables are for values that vary. HQ DNS is identical everywhere → plain static object. The package already gives "edit once, reinstall, everywhere gets it." A variable there adds zero capability and real cost: 2 a.m. indirection, unfilled-install risk, and an invitation for a rogue per-device override to silently break the invariant.

Not everything can carry a secret

Addresses, VIPs, IP poolsALL security profiles (web filter, AV, IPS, app ctrl…)
User groups, LDAP/RADIUS/TACACS+Services & groups, schedules, shapers
Metadata variables, dyn. cert, dyn. VPN tunnelSSL/SSH inspection, protocol options
Importing a per-box-different unsupported object = conflict (FMG must choose a value). Fix: uniquely named object, re-import. Exam distractor: "per-device map the web filter profile" — locked.

Business frame (say the money first)

"Junior's thirty copies work on day one and get more expensive every day after — every change becomes thirty changes, the first missed edit is silent drift, and the certified single baseline becomes thirty claims to verify by hand. Under per-device mapping, onboarding a branch costs one value in a table and the audited rules stay frozen."
