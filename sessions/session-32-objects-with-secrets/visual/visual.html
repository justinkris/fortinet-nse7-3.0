🎨 VISUAL LEARNING PAGE · SESSION 32 · DYNAMIC OBJECTS & METADATA VARIABLES

The Theatre of Objects

One script. Thirty actors. Opening night is install.

Recommended visual style

Real-world Analogy (Theatre Production) + Process Flow. This session's breakthrough came from the script/actor/opening-night scaffold — the exact analogy that unlocked the stalled derivation. Anchoring the visuals in that theatre metaphor, with one process-flow panel for the resolution pipeline and one decision panel for the static-vs-dynamic judgement, will bind the mechanism, the failure mode, and the design judgement to a single memorable scene. A plain topology diagram would waste the strongest memory hook the session produced.

Memory anchors to reinforce: ① the box gets the answer, not the question · ② install is a validation gate (fail loudly) · ③ variables are for values that vary · ④ some doors are locked (unique names).
1 · The One Script and the Thirty Actors

Purpose: cement the core mechanism — one certified package, per-device values held privately, substitution at install.
Image Generation Prompt
Flat minimalistic but highly detailed vector illustration, cream off-white background, coral accents, muted gold highlights, sage greens, soft brown outlines, clean bold line work, no gradients, no photorealism, storybook-professional style. A grand theatre stage viewed from the audience. Centre stage: a single enormous illuminated SCRIPT on a golden lectern, its cover stamped "BASELINE POLICY PACKAGE — CERTIFIED" with a wax seal shaped like the LoFinance logo (stacks of banknotes frozen inside an ice cube). One visible script line reads "allow LAN → $(branch_lan)". Fanning out across the stage in two curved rows: thirty small identical actor figures dressed as branch-office firewalls (rectangular sage-green appliance bodies with tiny status LEDs, little legs and arms), each holding a small private cue card close to their chest. Three cue cards are legible in the front row: "172.16.1.0/24", "172.16.2.0/24", "192.168.5.0/24" — each different. Stage-left, a director's chair holds Señor: older engineer, wide sombrero, immaculate posture, chaotic desk beside him piled with mandarin peels, holding a megaphone labelled "INSTALL". Warm coral stage lighting beams connect the script to each actor, and along each beam the placeholder text "$(branch_lan)" visibly transforms mid-beam into that actor's private subnet. In the wings, a young overconfident engineer (Junior) drags a wheelbarrow overflowing with thirty photocopied scripts toward a bin marked "DRIFT". Visual hierarchy: script largest, beams second, actors third. Do NOT include: dense technical labels, vendor logos, gradients, 3D rendering. Learning purpose: the package is the single frozen script; each device's truth is a private line; install is the moment of substitution.
Accessibility: a theatre stage where one master script projects light beams to thirty small firewall actors, each beam converting a placeholder into that actor's unique subnet, while a mentor in a sombrero directs and a junior wheels thirty rejected photocopies to a bin.
2 · Opening Night: The Substitution Pipeline
[ IMAGE PLACEHOLDER — 16:9 ]
Aspect ratio: 16:9 · Placement: after the mechanism explanation
Purpose: fix the resolution order and timing — generic name → FMG lookup → per-device wins over default → preview → resolved value lands on the box.
Image Generation Prompt
Flat minimalistic detailed vector illustration, cream background, coral/gold/sage palette, soft brown outlines, no gradients, no 3D. A left-to-right backstage process flow rendered as five theatre "stations" connected by a coral carpet. Station 1 "THE SCRIPT": a page close-up showing the line "set subnet $(branch_lan)" with a magnifying-glass-with-$ icon hovering beside it. Station 2 "THE PROMPTER'S BOOTH" (label: FORTIMANAGER): a cosy wooden booth where a prompter consults a ledger with two columns — a faint grey row "default: (empty)" and a bold gold row "Branch-17: 172.16.17.0/24"; the gold row is circled with a note "specific beats general". Station 3 "DRESS REHEARSAL" (label: INSTALL PREVIEW): a mirror in which the placeholder text is already shown transformed into "172.16.17.0/24" — the mirror labelled "the preview is where secrets confess". Station 4 "THE VALIDATION GATE": a sturdy stage door with a bouncer figure holding a clipboard, stamp reading "RESOLVED?"; a green tick lane passes through, and a small side lane shows a script page with an UNFILLED blank being turned away with a red "INSTALL FAILED" stamp — the rejected page looks relieved, not tragic. Station 5 "THE PERFORMANCE" (label: FORTIGATE): a single spotlit firewall appliance on stage speaking only the final line "172.16.17.0/24" in a speech bubble; a small caption beneath it: "the box gets the answer, not the question". Learning purpose: substitution happens inside FMG at install; per-device beats default; unresolved = loud failure; the device stores only resolved values.
Accessibility: a five-stage backstage pipeline showing a placeholder line travelling from script, through a prompter's ledger where the per-device value wins, past a preview mirror and a validation gate that rejects unfilled blanks, to a firewall on stage speaking only the final resolved subnet.
3 · The Boring Prop: When Not to Abstract
[ IMAGE PLACEHOLDER — 3:2 ]
Aspect ratio: 3:2 · Placement: beside the static-vs-dynamic section
Purpose: anchor the session's judgement lesson — invariant values take a static object; needless indirection is pure cost.
Image Generation Prompt
Flat minimalistic detailed vector illustration, cream background, coral/gold/sage palette, soft brown outlines, no gradients, no 3D. A split two-panel theatre props-room scene. LEFT PANEL, warm and calm, titled "THE INVARIANT": a plain, sturdy, slightly boring wooden signpost prop reading "HQ DNS — 10.10.10.53", bolted to the floor with four solid bolts, a small brass plaque underneath: "same line, every show, forever". Señor stands beside it in his sombrero giving a small approving nod, one hand resting on the sign, a half-peeled mandarin in the other. RIGHT PANEL, subtly chaotic, titled "JUNIOR'S VERSION": the same sign rebuilt as an elaborate contraption — a placeholder banner "$(hq_dns)" suspended by thirty tangled strings, each string running to an identical tiny cue card that ALL read "10.10.10.53"; one string is snapped and its card blank (tagged "install fails"); another card has been secretly scribbled to a different address by a mischievous hand reaching in from off-frame (tagged "silent drift"); Junior stands proudly beside it holding a hammer with a gift ribbon still attached, labelled "NEW HAMMER". A single caption band across the bottom of both panels: "Variables are for values that vary." Colour cue: left panel dominated by stable sage and gold; right panel infected with tangled coral string. Do NOT make Junior look mocked — earnest, not ridiculed. Learning purpose: identical-everywhere values belong in a static object; abstraction without variance buys risk and buys nothing.
Accessibility: two panels — a simple bolted-down sign holding one permanent DNS address approved by the mentor, versus the same sign rebuilt as thirty tangled strings holding identical cards, one snapped and one secretly altered, built by a proud junior holding a brand-new hammer.
4 · The Locked Dressing Rooms
[ IMAGE PLACEHOLDER — 3:2 ]
Aspect ratio: 3:2 · Placement: beside the per-device support matrix
Purpose: make the support matrix memorable — some object types simply cannot carry a per-device secret; there, the name itself must change.
Image Generation Prompt
Flat minimalistic detailed vector illustration, cream background, coral/gold/sage palette, soft brown outlines, no gradients, no 3D. A backstage corridor of theatre dressing-room doors receding in gentle perspective. LEFT SIDE: doors standing OPEN with warm gold light spilling out, each door's star-plaque naming a per-device-mappable type: "ADDRESSES", "VIPs", "IP POOLS", "USER GROUPS", "LDAP / RADIUS", "METADATA VARIABLES"; inside the nearest open door, one actor is visibly swapping between several private cue cards on a mirror. RIGHT SIDE: doors firmly SHUT with heavy brass padlocks and a polite sage-green sign "NO PER-DEVICE MAPPING", plaques reading "WEB FILTER", "ANTIVIRUS", "IPS", "SERVICES", "SCHEDULES". In front of the locked WEB FILTER door, Cipher (security engineer, restrained posture, dark practical clothing, unimpressed expression, coffee cup with tiny ice crystals forming on its surface) hand-letters two SEPARATE name placards — "WebFilter-BranchA" and "WebFilter-BranchB" — and hangs them on two adjacent plain doors, with a small resigned caption: "when the door is locked, the NAME must change." At the corridor's end, a small warning poster shows two arrows colliding, captioned "import per-box differences here → CONFLICT". Visual hierarchy: open-vs-locked contrast dominant. Do NOT overcrowd with every object type — six or seven plaques per side maximum. Learning purpose: security profiles, services and schedules cannot per-device map; unique names are the only mechanism, and importing per-box differences of unsupported types forces a conflict.
Accessibility: a backstage corridor where dressing-room doors for addresses and user groups stand open and glowing, while doors for web filter, antivirus and services are padlocked, and a security engineer hangs two uniquely named placards on plain doors instead.
5 · One Law, Three Costumes

Aspect ratio: 4:3 · Placement: end of page, consolidation panel
Purpose: compress three install-failure modes (S31 + S32) into the single validation-gate law — the session's most exam-proof consolidation.
Image Generation Prompt
Flat minimalistic detailed vector illustration, cream background, coral/gold/sage palette, soft brown outlines, no gradients, no 3D. A single imposing stage door centre-frame, styled as a friendly but immovable VALIDATION GATE: solid timber, brass fittings, a large gold stamp mechanism above it reading "RESOLVED? → ENTER", and a warm green lamp glowing over the pass lane. Queuing before the gate, three theatrical characters in different costumes, each politely but firmly turned away by the same calm gatekeeper (a small owl-like stage manager with a clipboard): FIRST, a script page costume whose line reads "port3" with a tag "no such name on this box" (missing mapping); SECOND, a pair of near-identical twin costumes labelled "WAN" and "wan" standing back to back, unaware of each other, tag "case mismatch — two different objects"; THIRD, a costume with a conspicuously EMPTY speech bubble and a blank cue card, tag "unfilled variable". Behind the gate, glimpsed through the open pass lane, a spotlit stage where a fourth character with a fully written card "172.16.17.0/24" performs happily. A banner across the top of the frame: "INSTALL IS A VALIDATION GATE, NOT A DELIVERY VAN". A small footer ribbon: "fail loudly beats succeed wrongly". Mood: reassuring rather than punitive — the gate is protective, the rejected characters look mildly sheepish, not distressed. Do NOT include error-red panic tones; use coral sparingly for the rejection tags only. Learning purpose: missing mapping, case mismatch and unfilled variable are one machine — install validates every reference against FMG's knowledge of the target and aborts rather than pushing a guess.
Accessibility: a protective stage door where a calm gatekeeper turns away three costumed failure modes — a wrong interface name, case-mismatched twins WAN and wan, and an empty speech bubble — while a fully resolved value performs on the stage beyond.
