SESSION 06 · 5-MIN BITE

One Box, Many Firewalls

The Core Idea

One FortiGate → up to 10 (default) independent virtual firewalls (VDOMs), each with isolated policy, interfaces, and security config, sharing the same hardware. Same trick as a VRF, one layer up.

Global VDOM ≠ a firewall

Super-Admin-only device-wide admin view (firmware, interfaces, logging). Excludes Policy & Objects, VPN, and User & Authentication — those are inherently per-VDOM concepts.

Exam GotchaTransparent VDOM still needs ONE manageip for admin access — "no routing" ≠ "no IP at all."

Management VDOM

Root is management VDOM by default (changeable): handles FortiGuard updates, license validation, admin access, FortiManager. Losing its connectivity can affect FortiGuard for ALL VDOMs.

Resource Limits

Default: 10 VDOMs per FortiGate, extendable via licensing. Global Resources (whole-device) vs per-VDOM resources (Override Maximum ≥ current usage) govern shared CPU/memory/session table.

Remember This

  • VDOM = VRF for firewalls
  • Global VDOM ≠ traffic-passing VDOM
  • Admin VDOM: max 1, zero traffic
  • Transparent still needs a manageip
  • 10-VDOM limit is licensable, not fixed
