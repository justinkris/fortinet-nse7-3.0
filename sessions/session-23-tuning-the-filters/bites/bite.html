Tuning the Filters

Web filter · App control · ISDB · FortiGuard dependencies

The one insight

Web filter is a live conversation with FortiGuard. App control and ISDB are stored knowledge that lives on the FortiGate. When the FortiGuard link (or the licence) goes down, they fail in different directions — and that asymmetry is your diagnostic key.

Asymmetric failure

FeatureData livesWhen FDS down
Web filterFortiGuard servers (cache local)Cache-miss hangs. Degrades fast.
App controlLocal databaseRuns on stale signatures. Degrades slowly.
ISDBLocal databaseRuns on stale vendor IPs. Degrades slowly.
Remember

Local = survives the outage. On-demand = dies with it. License expired ≠ feature broken.

The cache

FortiGate caches recent web-filter verdicts. This masks a FortiGuard outage temporarily. But the cache is a decaying asset — entries age out, new sites always miss, hit rate slopes down monotonically. A silent, gradual outage is what you see.

Fail-open ≠ instant permit

Fail-open governs the verdict. Timeouts govern the experience. Fail-open (the default) means "permit AFTER giving up waiting." The user's browser runs out of patience before FortiGate's request timeout expires. Result: hang, even under fail-open.

Three commands you must know

  • diagnose autoupdate versions — paperwork (Version, Contract Expiry, Last Update, Result). Web filter absent from this output.
  • diagnose debug rating — live conversation with FortiGuard rating servers. Watch for Status: Disable, RTT: -1, Flags: DI.
  • diagnose webfilter fortiguard cache dump — current belief. What verdicts does the box hold right now?
Never use execute update-now as a diagnostic. It's an action, not a read. It destroys the evidence.

Where each thing lives in config

  • Web filter: config webfilter profile → config ftgd-wf → config filters
  • App control: config application list → config entries
  • ISDB: config firewall internet-service-name — then referenced ON the policy as internet-service-name, not as a profile
  • FMG-as-FDS: config system central-management → set include-default-servers disable + rating server entry

The Penny translation

"The service we pay Fortinet for stopped answering our calls. The other two things on the invoice kept working because they live on our firewall, not on Fortinet's servers. The renewal is the licence. The FortiManager work is the insurance."
