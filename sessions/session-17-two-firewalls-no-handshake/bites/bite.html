5-Minute Revision · Session 17

Two Firewalls, No Handshake

Fault 1

IPsec Phase 2 Never Establishes

sa=0, rxp=txp=0, proxyid_num=1/child_num=0 = Phase 1 up, Phase 2 dead. Bilateral IKE debug shows NO_PROPOSAL_CHOSEN — compare both logged proposals to find the differing line.

Fix

Mismatched dhgrp under phase2-interface. Raise the weaker end, don't downgrade the stronger.

Fault 2

OSPF Area Mismatch → Silent Discard

Interfaces up, POINTOPOINT, matching cost, hellos flowing — table still empty. Area stamp is checked before any other OSPF processing; mismatch = discarded at the door, no Init entry ever forms.

Fix

Link must be Area 0 — a redundant link between two Area-0 sites can't sit in a non-backbone area, or you risk a backbone partition the moment the primary path dies.

Fault 3

The Static Route That Lied

Adjacency Full, LSDB correct, but routing-table ospf shows nothing for the prefix. Check routing-table all: a static route (AD 10) is silently beating OSPF (AD 110) for the same prefix.

Fix

Disable the static. Redistributing it into OSPF would NOT have worked — redistribution advertises, it never deletes the source route.

The One Rule Tying It Together

Full adjacency + correct LSDB ≠ correct routing table. Every fault this session lived one layer below where the previous check said "healthy." Always verify the actual winning route, not just the protocol's calculation.
