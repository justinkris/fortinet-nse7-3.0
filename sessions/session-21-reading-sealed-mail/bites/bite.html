Reading Sealed Mail

The go-live disaster fingerprint

Every site + every user + every browser warns = the inspection CA is missing from client trust stores. The forging engine works; the clients don't trust the forger. Evidence: cert viewer → issuer = your FortiGate CA, issued minutes ago. Order of operations IS the deployment plan: CA first, inspection second. Recovery: fall back to certificate inspection (needs no client trust) — never roll back to nothing.

Exemptions — permanent, not workarounds

Exempt traffic gets zero inspection of any depth; FortiGate steps out entirely. Built-in profiles are read-only → clone. Fortinet pre-exempts Finance, Health, Personal Privacy categories. Three permanent reasons: ① compliance/data custody (can't hold plaintext credentials), ② refuse the capability (admins shouldn't be able to see it), ③ certificate pinning.

Pinning — the wax seal

Pin lives in the app, compiled-in. Browser asks "do I trust the signer?" App asks "is this literally the cert/key I was born knowing?" So: CA pushed and trusted → browser fine, app still fails. Trusted ≠ pinned. Pinned apps break under full inspection no matter what's in the trust store.

Whose private key? — inbound vs outbound

Full inspection always requires a private key. Outbound: can't own Google's key → forge with re-signing CA → all the client pain → scope = Multiple Clients Connecting to Multiple Servers. Inbound: you own the server → load its real cert + private key → zero client impact, pinning irrelevant → scope = Protecting SSL Server (set server-cert-mode replace).

Protocol port mapping — the multi-select

Proxy engine dispatches to protocol inspectors by destination port, before decryption. SMTPS default = 587 only; port 10587 is never claimed even with full inspection on. Fix: config smtps → set ports 587 10587. Three preconditions, ALL required: full inspection + Multiple-Clients-to-Multiple-Servers + PROXY mode. Flow-based inspects all ports regardless — the evasion fails against flow. The costume room is proxy; the window is flow.

TLS version control

config https
  set unsupported-ssl-version block
  set min-allowed-ssl-version tls-1.2
  • CLI-only (exam bait). Also available for SMTPS, POP3S, IMAPS, FTPS.
  • Blocks at the handshake — before certificates. Cert-inspection fallback won't fix it.
  • Server offers 1.1 + 1.2, minimum 1.2 → allowed (negotiates 1.2).
  • Legacy devices: never lower the global minimum. Scoped exception pinned at both ends + logged + expiry date. "An exception without an expiry is the new normal."

Missed-question repair kit

SNI/SAN (third miss): SAN = Single cert, All the Names. Many individual certs needing selection = SNI. Find the count in the stem first.
Cert-inspection HTTPS protection = "Limited" — standalone. Not "No", not conditional on IPS (IPS needs payload, which cert inspection never provides).
