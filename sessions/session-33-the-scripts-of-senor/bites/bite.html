The LawThe change exists only where it landed. Downstream needs deliberate delivery; upstream needs deliberate import.

The Three Traps

  • Green checkmark ≠ delivered. It means "ran." Junior's admin profile sat Modified in the mirror while Riley couldn't log in. Cure: Install Device Settings.
  • Policies never climb alone. Remote-script policy changes reach the device DB via auto-retrieve but require manual Import for the package. Device-DB policy edits cannot auto-import — ever.
  • Don't import after a package script. Scripts write to the source of truth; a careless import drags the stale device-layer view back over your work.
Why the Import gate exists: the device DB mirrors one box (blast radius: one — safe to automate). The package is shared truth for the fleet (blast radius: everyone — a human must open the gate).

CLI vs Tcl

  • CLI script = a letter. Fixed FortiOS commands, identical for every recipient. Familiar syntax + metadata variables. Cannot decide — and cannot invent new mistakes.
  • Tcl script = a person. proc, if/else, loops — a program that generates commands and can branch per device (the Melbourne port2 box). Pays for logic with complexity and 2am auditability.

Two Anchors Worth the Whole Session

"Reinstall skips the questions, never the confession." Reinstall Policy skips the wizard (bookkeeping) but keeps the preview (safety) — and pushes device-DB modifications too, so the preview matters more.
"Automation removes variance, not mistakes." — Cipher. A wrong change also ships before lunch. Staged changes + out-of-band SSH fixes = reconcile revision history before installing.
