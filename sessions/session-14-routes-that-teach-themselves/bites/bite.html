SESSION 14 · BITE · OSPF OVERVIEW

Routes That Teach Themselves

5-minute revision — link-state routing, areas, ABR/ASBR/backbone.

THE PROBLEM

Why Static Routing Fails

A static route only knows what a human typed. No topology awareness, no "is there a better path now?" — and it can be perfectly configured and still cause an outage (Parramatta: route vanished, traffic fell through to the default route unencrypted). Gets worse with every new site.

THE FIX

Link-State Routing (OSPF)

  • Distance-vector ("routing by rumour") — trusts secondhand info from neighbors. Chinese-whispers risk.
  • Link-state (OSPF) — every router builds its own identical copy of the raw topology (the LSDB) and runs Dijkstra's algorithm independently. Same map → always-consistent results, even though each router's tree is rooted at itself.
  • Quiet by design: full refresh every 30 minutes; triggered updates only on real topology changes.
SCALING

Areas — Containing the Blast Radius

One flat LSDB doesn't scale: size grows unbounded, and one link flap anywhere forces a recalculation everywhere. Areas fix both — the ABR keeps a full LSDB per area it touches, but exports only a Type 3 summary LSA (prefix + cost) across boundaries, discarding internal detail.

Watch for: "ABR tracks LSAs across areas" is imprecise — it maintains a SEPARATE full LSDB per connected area.
THE RULE

Backbone Area (0.0.0.0)

Every area must connect to Area 0 — no direct area-to-area summary exchange. Why: summaries discard detail, so loop detection is impossible. OSPF prevents the dangerous topology shape instead of trying to detect corruption after the fact.

ROLES

Router Roles Are Compound

  • Internal router — one area, one LSDB.
  • Backbone router — ≥1 interface in Area 0.
  • ABR — interfaces in multiple areas, always touches backbone.
  • ASBR — imports non-OSPF routes via redistribution.
🧠
MENTAL NOTE
One router can be ABR + backbone router + ASBR simultaneously. Check EVERY definition — don't stop at the first match.
