TECHNICAL CONCEPTS COVERED
------------------------------------------------------------------
1. STITCH = TRIGGER + ACTION(S). If/then automated workflow.
   Fabric NOT required for stitches; but in a fabric, stitches are
   configured on the ROOT, scoped to all or subset of FGTs, and
   PUSHED to relevant leaves.
2. PUSH AT CONFIG TIME, NOT TRIGGER TIME. Leaves own local copies;
   root does NOT need to be operational for previously configured
   stitches to run on leaves. (Exam favourite. Rhymes with S36
   "crown vacant, witnesses keep writing.")
3. USE CASE 1 — HIGH CPU: preconfigured High CPU trigger; config
   system global / set cpu-use-threshold (default 90). Two custom
   actions: Collect_Diagnostics_Action (CLI script) then
   Email_Diagnostics_Action with %%results%% as email body.
   SEQUENTIAL mode + 30s DELAY so the script COMPLETES before the
   email consumes its output (delay = dependency, not courtesy).
4. USE CASE 2 — CONFIG BACKUP: SCHEDULE trigger (time itself is
   the trigger — no verb-less stitch). Frequencies: once, hourly,
   daily, weekly, monthly; GUI menu changes with frequency choice.
   ROOT FortiGate's clock governs. Results land in
   Configuration > Revisions; usable for rollback. All-or-subset
   FGT scoping.
5. USE CASE 3 — IoC DETECTION + AUTOMATED QUARANTINE. Required 4:
   FortiGate, FortiAnalyzer, FortiClient EMS, FortiClient.
   FortiClient connects to BOTH EMS and FGT; FC + FGT both log to
   FAZ; FGT preconfigured with EMS IP + admin credentials.
   5 steps: (1) FortiClient logs malicious site → FAZ;
   (2) FAZ discovers IoC → notifies FGT (the SCRIBE is the
   DETECTIVE — detection lives where memory lives);
   (3) FGT verifies endpoint is a CONNECTED FortiClient endpoint
   → notifies EMS; (4) EMS locates endpoint → sends quarantine
   message; (5) endpoint quarantines ITSELF, blocks all traffic,
   reports status to FGT + EMS (loop closes).
   Manual trigger CLI: diagnose endpoint fctems
   queue-complete-calls Q-<endpoint_ip>. NOT supported on
   FortiClient (Linux).
6. WHY SELF-QUARANTINE BEATS FIREWALL BLOCK: "You can only block
   what is inline" (STUDENT'S OWN SENTENCE — canon). Firewall
   never sees same-L2-domain traffic; quarantine travels with a
   mobile endpoint; blocks the DEVICE not a DHCP-lease IP.
7. SECURITY RATING (intro only): fabric-wide audit vs Fortinet
   best practices, run from the fabric, score + prioritised
   actionable findings, some auto-fixable. Depth deferred to S39.
