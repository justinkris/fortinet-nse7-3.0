SESSION 38 · 5-MINUTE BITE · AUTOMATION STITCHES

If This, Then Firewall

1 · The Stitch

Trigger + action(s) = a stitch (if/then automation). Fabric not required — but in a fabric, stitches are configured on the root (Security Fabric > Automation), scoped to all or a subset of FGTs, and pushed to the leaves. Triggers can come from the FGT itself or other fabric devices via FAZ event handlers.

Exam gold: the push happens at config time, not trigger time — leaves hold local copies, so previously configured stitches keep running when the root is down. Root = author, not dispatcher.

2 · High-CPU Pipeline

Preconfigured High CPU trigger — threshold via config system global → set cpu-use-threshold, default 90%. Actions: CLI diagnostics script, then email whose body is %%results%% (the script's output).

Sequential + 30s delay — the email consumes the script's output, so the script must finish first. Simultaneous = empty envelope. Let the kettle boil before pouring.

3 · Scheduled Backup

Schedule trigger (time itself is the trigger): once / hourly / daily / weekly / monthly, governed by the root FortiGate's clock. Backups land in Configuration > Revisions — rollback-ready.

4 · IoC Quarantine (the 3am chain)

Required four: FortiGate · FortiAnalyzer · FortiClient EMS · FortiClient. FGT holds EMS IP + admin credentials.

1. FortiClient logs malicious site → FAZ 2. FAZ discovers IoC → notifies FGT 3. FGT verifies endpoint CONNECTED → notifies EMS 4. EMS sends quarantine message → endpoint 5. Endpoint quarantines ITSELF, reports back

Manual: diagnose endpoint fctems queue-complete-calls Q-<ip>. Not on FortiClient (Linux).

Why not a firewall block? "You can only block what is inline" — same-L2 traffic never crosses the firewall; the laptop is mobile; DHCP re-issues IPs. Only endpoint quarantine follows the compromise around.

5 · Security Rating (preview)

The fabric audits itself against best practices → score + prioritised findings, some auto-fixable. Watching (CPU) · Acting (quarantine) · Judging (rating). Full story: Session 39.
