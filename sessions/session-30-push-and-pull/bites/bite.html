Push and Pull — in 5 minutes

The four FortiManager verbs as directions, plus global policy packages. Enough to walk into the exam or the console.

Push overwrites. Pull observes. Pull first, look, then push.
The Pipeline

Three copies of truth

Everything is a direction across these three
[ POLICY LAYER ]  →  [ DEVICE DB ]  →  [ LIVE FORTIGATE ]
   intent             FMG's copy         the real box
The Verbs

Direction is the whole game

PUSH ↓ dangerous   PULL ↑ safe

Install is a blind overwrite. An undocumented direct-box edit that was never retrieved is destroyed by an unrelated install — no merge, no prompt.
The 214 Fix

A big diff is a confession

A preview showing far more than you changed = desync, not a fault. Reconcile before pushing:

Reconcile-before-push
RETRIEVE  box→DB   →   IMPORT  DB→policy (separate audit pkg)
   →  AUDIT the phantoms  →  re-apply your real changes  →  INSTALL w/ preview

Revert stages. Install delivers. Import reconciles. Revert alone changes nothing on the box.
