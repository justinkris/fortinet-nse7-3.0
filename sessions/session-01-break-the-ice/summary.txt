TECHNICAL CONCEPTS COVERED
--------------------------
1. Evolution of the enterprise network: perimeter-only security is dead; threats include
   zero-days, APTs, polymorphic malware, insider threats; malware enters via USB sticks /
   compromised personal devices; networks are no longer flat/one-dimensional.
2. Implicit trust inside the perimeter -> lateral movement. Zero-trust approach required.
3. Segmentation: only real if inter-segment traffic is FORCED THROUGH an inspection
   point (gateway on the firewall, not on a switch). Limits blast radius.
4. Port/IP filtering reads headers only ("envelopes"); in a 443-dominated world this is
   nearly worthless. Application-layer inspection reads the "letter."
5. NGFW = SUPERSET of traditional firewall: keeps ACLs/port-IP filtering/stateful
   inspection, ADDS application-layer analysis + UTM profiles (core security features).
6. UTM engines: antivirus/anti-malware (files), IPS (traffic exploits), web filtering,
   application control, advanced threat protection/sandboxing. SSL inspection is the
   ENABLER, not an engine — without decryption the engines inspect a locked box.
7. FortiGuard Labs: AI/ML-powered global threat intelligence feeding all engines in near
   real time. Config ≠ protection; engines are only as good as their feed.
8. Fortinet enterprise firewall solution: FortiGate 30-90 (branch), 100-900 (campus),
   1000-7000 (data center), FortiGate VM / FortiCNF / FortiSASE (cloud); ONE FortiOS
   everywhere; ONE management console (FortiManager). (The DEFW/ISFW/DCFW/NGFW role
   names were deliberately DEFERRED to Session 2 — Señor drew four boxes and walked away.)
