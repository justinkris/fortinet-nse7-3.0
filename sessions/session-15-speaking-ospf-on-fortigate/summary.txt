TECHNICAL CONCEPTS COVERED
---------------------------
1. OSPF NETWORK TYPES as SETS OF ASSUMPTIONS ABOUT THE MEDIUM (core mental model of the
   session — "config fails when assumptions and reality diverge; the CLI accepts the
   lie cheerfully"):
   - Broadcast: assumes shared medium where one multicast frame reaches everyone free
     (Ethernet). Runs DR/BDR election. 224.0.0.5 AllSPFRouters (hellos; updates FROM
     DR), 224.0.0.6 AllDRouters (updates TO DR/BDR).
   - Point-to-point: FortiGate DEFAULT on tunnel interfaces; assumes exactly ONE
     far-side neighbor. Correct for normal site-to-site tunnels.
   - Point-to-multipoint: for dial-up hub with net-device disable (one shared interface,
     many spokes). Treats interface as a bundle of individual P2P conversations;
     REPLICATES AND UNICASTS hellos/updates per spoke; NO DR/BDR ELECTION AT ALL; each
     spoke forms exactly one adjacency (with the hub); spokes learn each other's
     networks via hub flooding LSAs, never via each other's hellos.
2. WHY JUNIOR'S CONFIG PRODUCED TOTAL SILENCE (not flapping/half-formed): broadcast
   mode's multicast hello has no shared medium to deliver it and broadcast mode does
   not replicate per tunnel → hello dies at the interface → no hellos received → table
   EMPTY, not broken. Also broadcast would attempt a DR election among spokes who
   cannot hear each other's votes.
3. NEIGHBOR OUTPUT AS FORENSIC FINGERPRINT: `get router info ospf neighbor` role column
   reveals network type without reading config. Full/DR / Full/BDR / Full/DROther →
   election happened → broadcast. `Full/ -` → no election possible → P2P or P2MP.
   DROther is a CAMPAIGN RESULT (stood in an election and lost), not a neutral word for
   "ordinary neighbor" — student initially answered DROther for P2MP and this
   distinction landed hard. Exam trap explicitly flagged.
4. ADJACENCY STATE LADDER retrieval: Init → 2-Way → ExStart → Exchange → Loading →
   FULL; Full = identical LSDBs, healthy adjacency parks there. Student recalled Full
   cold.
5. REDISTRIBUTION: mechanism for injecting non-OSPF routes (static/connected/BGP/RIP/
   ISIS) into OSPF; performing it makes the router an ASBR. DEFAULT: ALL SOURCES
   DISABLED. Reasoned (not recalled) safety rationale built on Parramatta precedent:
   open-by-default would have flooded the emergency statics and unencrypted default
   route fleet-wide automatically — OSPF's superpower (automatic domain-wide trust) is
   exactly its blast radius. `redistribute static` is ALL-OR-NOTHING without filtering.
6. ROUTE FILTERING LADDER: access lists (simple prefix permit/deny) → prefix lists
   (adds ge/le mask-length operators; applies only to distribute-list-in) → route maps
   (can use access/prefix lists as match objects AND modify attributes). Design
   principle taught: choose the LEAST powerful tool that meets the requirement —
   "complexity you don't need is a standing tax on every future incident / paid at
   2 AM." Penny's one-prefix requirement → access list with exact-match.
7. COMPOUND ROUTER ROLES revisited on live topology: hub = ABR (Area 0 + Area 1) +
   ASBR (redistributes static) + backbone router (interface in Area 0). NOT internal
   (requires ALL interfaces in one area).

---------------------------

diagnostic evidence; (4) eliminated P2P by its one-neighbor assumption; (5) selected
P2MP by mechanism; (6) verified with neighbor output and read the fingerprint.
