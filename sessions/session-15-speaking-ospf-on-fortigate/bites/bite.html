Speaking OSPF on FortiGate

Network types = medium assumptions

TypeAssumesDR/BDRUse
BroadcastShared medium; one multicast reaches allElectedEthernet LAN
Point-to-pointExactly one far-side neighborNoneSite-to-site tunnel (FortiGate default)
Point-to-multipointMany spokes, no shared medium; per-spoke unicast replicationNeverDial-up hub, net-device disable
The rule: multiple dial-up spokes on one hub interface → hub interface must be point-to-multipoint.

Why broadcast on a hub = total silence

Broadcast mode multicasts its hello to 224.0.0.5 and expects the medium to deliver it to everyone. A dial-up hub interface has no shared wire — three separate tunnels — and broadcast mode never replicates per tunnel. The hello dies at the interface. No hellos → empty (not flapping) neighbor table. Bonus absurdity: it would try a DR election among spokes who can't hear each other's votes.

The fix + the fingerprint

config router ospf
    config ospf-interface
        edit "Hub"
            set interface "To-Spokes"
            set network-type point-to-multipoint
        end

Verify: get router info ospf neighbor → each spoke shows Full/ -.

  • Full = top of the ladder (Init → 2-Way → ExStart → Exchange → Loading → Full); identical LSDBs.
  • Dash = no election ran. DROther means "lost an election" — it can only appear on broadcast segments.
Fingerprint: the role column reveals the network type. DR/BDR/DROther → broadcast. Dash → P2P or P2MP.

Redistribution

  • Injects non-OSPF routes (static/connected/BGP/RIP/ISIS) into OSPF → router becomes an ASBR.
  • Default: disabled for every source — OSPF floods trust automatically domain-wide, so an open valve turns one careless static into everyone's route.
  • redistribute static is all-or-nothing without a filter.

2 AM tax: unneeded machinery must be audited during every future incident. One-sentence requirement → roughly one-line config.

Compound roles — check all four, every time

  • Internal: ALL interfaces in one area?
  • ABR: interfaces in multiple areas?
  • Backbone: any interface in Area 0?
  • ASBR: any non-OSPF source imported?

The LoFinance hub is ABR + backbone + ASBR simultaneously (not Internal). Under- and over-selecting are the same error: skipping the per-definition test.

Command toolbox

  • get router info ospf neighbor — states + roles
  • get router info ospf interface — per-interface, DR/BDR IDs
  • get router info ospf status — router ID, areas, full-adjacency count
  • get router info ospf database brief — LSDB by LSA type (Session 16)
  • get router info bfd neighbor — BFD status with OSPF
