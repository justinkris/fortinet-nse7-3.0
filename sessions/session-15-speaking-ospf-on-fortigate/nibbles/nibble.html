Speaking OSPF on FortiGate

The ruleDial-up hub, many spokes, one interface (net-device disable) → point-to-multipoint. Default on tunnels is point-to-point.
Why broadcast diesMulticast hello needs a shared medium to deliver it. No wire → hello dies at the interface → neighbor table empty, not flapping.
P2MP behaviourReplicates + unicasts per spoke. One adjacency per spoke (with the hub). No DR election, ever.
FingerprintFull/ - → P2P/P2MP (no election). Full/DROther → broadcast — DROther means "lost an election."
Full stateTop of the ladder: identical LSDBs on both sides. Healthy adjacencies park here.
RedistributionNon-OSPF routes into OSPF → box becomes ASBR. Default disabled for all sources. All-or-nothing without a filter.
Filter ladderAccess list (prefix) → prefix list (ge/le, distribute-list-in only) → route map (modifies attributes). Use the least powerful tool that works.
RolesOne box can be ABR + backbone + ASBR at once. Run all four definitions as independent checks — never count, always test.
