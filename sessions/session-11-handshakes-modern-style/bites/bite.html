⚡ Session 11 Bite — Handshakes, Modern Style

IKEv2 exchanges · IKEv1 comparison · network-id — 5 minutes.

The three jobs, in order

① Agree the language (proposals) → ② Grow the shared secret (Diffie-Hellman — the key is grown at both ends, never sent) → ③ Prove identity.
"First grow the secret. Then whisper your name inside it." Identity goes after DH — before DH, everything is plaintext.

Why aggressive mode leaks

It sends peer IDs in the first messages — before DH — to save round trips. Its speed and its clear-text ID leak are the same design choice, not two facts.

IKE_SA_INIT (2 msgs): proposals + DH publics — plaintext but nothing sensitive. IKE_AUTH (2 msgs): identities + auth + first child SA piggybacked — encrypted. No separate quick mode. Modern kit: AES-GCM, SHA-2, ECP DH groups 19/20, EAP.

network-id

Fortinet-proprietary · IKEv2-only. Selects the correct phase 1 so multiple tunnels can exist between the same local/remote gateway pair. Must match at both ends. Nightclub with two identical doors — the ticket names the event.
set ike-version 2
set network-overlay enable
set network-id 1        # tunnel 2 gets network-id 2

Exam traps

• "Aggressive skips DH" — false, it reorders. • "PSK sent in clear" — false, the IDs are. • IKEv2 = 4 msgs, not 2, not 6. • network-id on IKEv1 = always false. • network-overlay enable AND network-id N — two commands.

The senior lesson

"Fast or safe?" was a false dilemma — IKEv2 beat aggressive on speed AND main mode on economy. When offered a trade-off, first check it's real. Refuse the question.
