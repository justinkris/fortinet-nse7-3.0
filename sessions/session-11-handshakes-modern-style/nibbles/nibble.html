⏱ 60-Second Nibble · Session 11

The ordering ruleDH first, then identity — before DH everything is plaintext. "Grow the secret, then whisper your name inside it."
Aggressive modeFast because it sends peer IDs before DH → IDs in clear text. Speed and leak = one design choice. 3+3 = 6 msgs.
Main modeSafe but chatty: 6+3 = 9 msgs, IDs encrypted.
IKEv24 msgs: IKE_SA_INIT (proposals + DH publics) + IKE_AUTH (encrypted identities + first child SA piggybacked). No quick mode. AES-GCM, SHA-2, ECP groups, EAP.
network-idFortinet-proprietary, IKEv2-only. Multiple tunnels between the same gateway pair — the event printed on the nightclub ticket. network-overlay enable + network-id N, matching both ends.
"Fast or safe?" — refuse the question. IKEv2 is both.
