TECHNICAL CONCEPTS COVERED
---------------------------
1. THE PIPELINE (session's spine): LSDB → Dijkstra → SPF tree → OSPF routes → routing
   table. Routing table = "last exhaust of the pipe."
2. WHY THE MAP EARNS ITS MEMORY: independence. A link-state router recomputes locally
   the moment it learns a FACT changed; a distance-vector router waits on daisy-chained
   RUMOUR ("routing by rumour" — student's S14 coinage, retrieved cleanly). Señor's
   wall quote: "Rumour spreads answers. Link-state spreads facts. Every router computes
   its own answers from facts." Tied to Parramatta outage precedent.
3. LSAs: routers advertise their LINKS (firsthand facts: connected networks + interface
   costs), never routes/answers. LSDB = collection of all LSAs (own + received). LSAs
   are acknowledged. Link state update packet = OSPF header + string of LSAs, each with
   its own header.
4. QUIETNESS + TERMINOLOGY FILED: periodic re-advertisement every 30 MINUTES; between
   refreshes, silence unless a topology change causes a TRIGGERED UPDATE (a.k.a.
   triggered/incremental LSA flooding). This closed the S13/S14 naming-recall item —
   student had the mechanism for two sessions; the label is now attached. (But see Q4
   miss below — the 30-min number needs one more rep under exam pressure.)
5. COST: 16-bit, 1–65,535, lowest TOTAL path cost wins; route cost = sum of EXIT
   interface costs along the path. Worked example from study guide p.313–314 (R1–R5,
   Net1–Net5): student computed via-R3 = 1+2 = 3 vs via-R4 = 3+1+2 = 6, correct.
   NOTE: student asked for the topology to be re-presented TWICE (first as redrawn
   ASCII, then as a segment-connection table) before answering — the TABLE format
   (segment | who connects | cost each pays) unlocked it instantly. Prefer
   connection-tables over ASCII art for this student's topology reasoning.
6. STUDENT-DRIVEN TANGENT (high value): "what if costs differ on each end of a link?"
   → COST IS DIRECTIONAL, paid on exit only; asymmetric costs across parallel paths
   create ASYMMETRIC ROUTING with every routing table technically correct — which
   breaks STATEFUL FIREWALLS (HA sessions callback: SYN out one box, SYN-ACK back via
   another, no session state, drop). Señor margin quote: "Cost is paid on exit.
   Symmetry is your job, not OSPF's." STUDENT EXPLICITLY REQUESTED this scenario be a
   dedicated section in the study guide — done (see complete guide, "The Asymmetric
   Cost Trap"). This was genuinely senior-grade curiosity.
7. SPF TREE: Dijkstra iterates per destination, keeps cheapest, repeats until every
   network/router has one winning branch; local router at root; destinations can be
   networks OR routers.
8. SAME MAP, DIFFERENT TREES (S14 knot revisited deliberately): LSDB identical per
   area; TREES differ (each rooted at its own router); no contradictions because all
   trees derive from the same facts — per-hop decisions dovetail. Loops in link-state
   = symptom of UNSYNCHRONISED LSDBs. Student answered "still the same tree, but built
   from the router as the root" — corrected: the trees genuinely DIFFER; what's
   identical is the MAP. One-word polish, concept solid.
9. FULL re-certified with new depth: Full = identical synchronised LSDBs between
   neighbours = the moment routing through each other becomes a safe bet. Adjacency
   below Full = a neighbour whose map you can't vouch for.
10. AREAS: 32-bit ID, decimal (10) or dotted (0.0.0.10) notation — same number, exam
    sleight-of-hand. SEPARATE LSDB per area; identical-databases premise is PER AREA.
    Internal router = 1 LSDB; multi-area router = one FULL LSDB per connected area.
    Benefits: smaller DBs, contained blast radius (flood + SPF recalcs stay in-area),
    summarisation at borders. Trade-off: design planning + harder troubleshooting
    ("a boundary you draw is a boundary you own at 2 AM"). Backbone = Area 0.0.0.0,
    mandatory, all areas connect to it hub-and-spoke (physical or virtual link).
11. FLAT-LSDB TWO COSTS retrieval (from S14): student retrieved (a) unbounded DB
    size/memory/CPU including weak branch boxes — clean; needed one nudge for
